## 二、发现的 Bug

### 🔴 P1-1：`sendSms()` 调用不存在的云端端点 — 短信功能静默失效

**文件：** `background.js` 第 114-137 行

```javascript
async function sendSms(phone, tabId) {
  if (await isPcAlive()) {
    try { const r = await fetch(`${PC_BASE}/sms?...`); if (r.ok) return; } catch {}
  }
  const pin = await getPin();
  if (!pin) { notifyTab(...); return; }
  try {
    await fetch(`${await getCloudApi()}/api/v1/sms`, {   // ← 云端没有这个端点
      method: 'POST',                                      // ← process_request 不支持 POST body
      headers: { 'Content-Type': 'application/json', 'X-AutoDial-PIN': pin },
      body: JSON.stringify({ phone }),
      signal: AbortSignal.timeout(5000)
    });
  } catch {}   // ← 错误被吞，用户无任何反馈
}
```

**问题：**
1. 云端 `cloud_relay_v2.py` 只实现了 `/api/v1/dial`、`/api/v1/hangup`、`/api/v1/status`，**没有 `/api/v1/sms`**
2. 即便有，`process_request` 不接收 POST body（复查版 P0 已确认），POST 请求的 JSON body 无法读取
3. `catch {}` 吞掉所有错误，用户不知道短信没发出

**影响：** PC 离线时，短信功能完全失效且无提示。PC 在线时正常（走 localhost `/sms`）。

**修复方案（二选一）：**

A. 云端补 `/api/v1/sms` 端点（GET + Header PIN + query number/content），与 dial 同模式：
```python
if path == '/api/v1/sms':
    pin = hdrs.get('x-autodial-pin', '')
    number = parse_qs(parsed.query).get('number', [''])[0]
    content = parse_qs(parsed.query).get('content', [''])[0]
    # ... 校验 + ensure_future 转发 {'type':'sms','number':number,'content':content}
```
background.js 改为 GET：
```javascript
await fetch(`${await getCloudApi()}/api/v1/sms?number=${encodeURIComponent(phone)}`, {
  headers: { 'X-AutoDial-PIN': pin }
});
```

B. 如果短信不需要云端兜底，去掉 background.js 第 126-136 行的云端分支，PC 离线时明确提示"短信需要 PC 端在线"。

### 🟡 P2-1：`hdrs` 变量在异常路径下未定义 — 边缘 NameError

**文件：** `cloud_relay_v2.py` 第 796-871 行

```python
try:
    hdrs = dict(request_headers)        # ← 如果这行抛异常
    upgrade = hdrs.get('upgrade', '')
    if upgrade == 'websocket':
        return None
except Exception as e:
    log.warning(f'WS_CHECK_FAIL: {e}')
    try:
        for key in request_headers:     # ← fallback 不定义 hdrs
            ...
    except Exception:
        pass

parsed = urlparse(path)
path = parsed.path
...
if path == '/api/v1/dial':
    pin = hdrs.get('x-autodial-pin', '')  # ← hdrs 可能未定义 → NameError
```

**触发条件：** `dict(request_headers)` 抛异常。正常情况下几乎不会发生（request_headers 是 Headers 对象，支持 dict() 转换），但防御性编程应处理。

**修复：** 在 except 块中补 `hdrs = {}`，或在 try 之前初始化：
```python
hdrs = {}
try:
    hdrs = dict(request_headers)
    ...
```

### 🟡 P2-2：`pcAvailable` 缓存在 PC 变不可达后不重置 — 每次拨号多等 2 秒

**文件：** `background.js` 第 55-66 行

```javascript
async function dial(phone, tabId) {
  if (await isPcAlive()) {          // ← 缓存 true，不会重新检测
    try {
      const res = await fetch(`${PC_BASE}/dial?...`, { signal: AbortSignal.timeout(2000) });
      if (res.ok) { notifyTab(...); return; }
    } catch {}                       // ← PC 超时/拒绝，pcAvailable 仍为 true
  }
  // 走云端
}
```

**场景：** PC 先在线（pcAvailable 缓存为 true）→ PC 关闭 → 用户拨号 → `isPcAlive()` 返回缓存的 true → `fetch` 等 2 秒超时 → catch → 走云端。**下次拨号同样**，每次白等 2 秒，直到页面刷新触发 `checkPc` 重置。

**修复：** 在 catch 中重置缓存：
```javascript
} catch {
  pcAvailable = false;   // PC 已不可达，重置缓存
}
```

### 🟡 P2-3：`record_message('rest_dial', 64)` 统计分类错误

**文件：** `cloud_relay_v2.py` 第 902 行

```python
def record_message(pin, msg_type, bytes_count):
    if msg_type in ('dial', 'sms', 'hangup'):
        total_bytes_sent += bytes_count
    else:
        total_bytes_received += bytes_count   # ← 'rest_dial' 走这里，计为"接收"
```

REST 拨号是服务器**发送**给手机的，应计入 `total_bytes_sent`。但 `msg_type='rest_dial'` 不在 `('dial','sms','hangup')` 中，被误计入 `total_bytes_received`。

**修复：** 改为 `record_message(pin, 'dial', 64)` 或在 record_message 的判断中加入 `'rest_dial'`。

### 🟡 P2-4：manifest 缺 `clipboardWrite` 权限

**文件：** `manifest.json`

原 11pin 版有 `"clipboardWrite"`，整合版删了。但 content-script 第 647 行用了 `navigator.clipboard.writeText()`。在 HTTPS 页面可能可用，但加上权限更保险。

**修复：** permissions 加回 `"clipboardWrite"`：
```json
"permissions": ["activeTab", "storage", "clipboardWrite"],
```
