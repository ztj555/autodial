## 一、当前版本

| 组件 | 版本 | 认证 | 端口 |
|------|------|------|------|
| Chrome 扩展 | v4.x | X-AutoDial-PIN Header | HTTP 35432 (PC) / REST 35430 (云) |
| Electron PC | v3.x | PIN (手机号) | WS+HTTP 35432 |
| Go PC | v1.x | PIN (手机号) | WS+HTTP 35432 |
| 云中继 | v2 | PIN (4/11 位) | WS+HTTP 35430 |
| Android | v4.53 | PIN | WS 35430 (Cloud) |

> v3 JWT 端口 35440/35441 已废弃，全部统一到 35430。

---

## 二、连接路径矩阵

### 场景一：全本地（PC + 手机同网络）
```
扩展 → HTTP localhost:35432 → PC → WS 直连 → 手机
```
延迟 < 10ms，零外部依赖。

### 场景二：异地（扩展 + 手机不在同网络）
```
扩展 → REST 35430 → 云中继 → WS 35430 → 手机
```

### 场景三：混合（PC 连云，扩展走本地）
```
扩展 → HTTP localhost:35432 → PC → WS 35430（云中继）→ 手机
```
PC 作桥梁，最灵活的生产模式。

### 场景四：Go PC 替代 Electron
```
扩展 → HTTP localhost:35432 → Go PC → WS → 手机
```
协议完全兼容，扩展端透明。

## 四、PC 双实现差异

| 差异 | Electron | Go |
|------|----------|-----|
| 连接上限 | 2 台 | 10 台 |
| 界面窗口 | 4 窗 | 1 窗 |
| 短信窗口 | ✅ | ❌ |
| 安装体积 | ~150MB | ~10MB |

## 五、潜在风险

| # | 风险 | 严重度 | 备注 |
|---|------|--------|------|
| 1 | 手机同时连 LAN + Cloud 双路拨号 | 低 | ACK 去重 |
| 2 | Go 版 `sendToPhone` 多设备广播 | 中 | Electron 只发活跃设备 |
| 3 | 云端 fetch 无超时 | 低 | 公网部署前需补 |
