# AutoDial 浏览器扩展 v3.1

基于 Chrome Extension MV3，支持 PC 直连 + 云端双模拨号。手机号即账号，检测到即自动登录。

## 核心功能

1. **自动登录**：CRM 页面检测到坐席手机号 → 立即上报云端，后续拨号携带 JWT
2. **双模路由**：PC 直连优先（页面加载时检测一次，缓存复用），云端兜底
3. **浮动按钮**：CRM 页面注入可拖动的拨号/挂断按钮，8 套主题
4. **CRM 号码检测**：TreeWalker 扫描坐席手机号（顶层 frame）+ 客户手机号（子 iframe）
5. **右键菜单**：显示当前登录账号，未登录时红色"点击登录"一键扫号+登录

## 文件说明

| 文件 | 角色 |
|------|------|
| `manifest.json` | MV3 扩展清单，匹配 zhudaicms.com / rxhcrm.com / rongxinhui.com |
| `background.js` | Service Worker：JWT 管理 + 双模路由 + 拨号轮询 + PC 缓存检测 |
| `content-script.js` | 所有 frame 注入：TreeWalker 扫号 + 浮动按钮 + 主题 + 右键菜单 |
| `popup.html` + `popup.js` | 弹窗界面：服务器配置 + 手动登录 + 状态显示 |

## 安装

1. Chrome/Edge 打开 `chrome://extensions/`
2. 右上角开启"开发者模式"
3. 点击"加载已解压的扩展程序"
4. 选择 `AutoDial-Extension/` 目录

## 使用

### 首次使用

1. 点击扩展图标
2. 确认服务器地址（默认 `http://127.0.0.1:35441`，点击测试验证）
3. 打开 CRM 页面（guwen.zhudaicms.com）→ 插件自动检测手机号
4. 弹出框显示"检测到 153xxxxxxxxx，请点击登录"
5. 点击登录 → 完成

> 需要先启动服务端：Cloud Relay（35430）+ REST API（35441）。见 `cloud-relay/python/`。

### 日常使用

- 打开 CRM → 自动检测手机号 → 自动登录 → 点击拨号即可
- 登录状态在右键菜单底部可见
- 换号时自动重新登录，Token 过期静默续期

### 手动登录（自动检测失败时备选）

插件弹窗 → 手动输入手机号 → 点击登录

## 拨号按钮反馈

| 提示 | 原因 |
|------|------|
| `✓ 已拨出` | 拨号成功 |
| `✗ 未登录，请右键菜单登录` | 手机号未检测到或未登录 |
| `✗ 服务器不可达 (端口35441)` | REST API 服务器未启动 |

## 右键菜单

除主题切换、拨号、短信等功能外，底部显示登录状态：

- **已登录**：`👤 15397033187`（灰色不可点击）
- **未登录**：`🔴 点击登录`（红色加粗，点击即扫号+登录）
