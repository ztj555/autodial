# AutoDial 浏览器扩展

> MV3 | 最后更新：2026-07-18 | PC 直连优先 → 云端兜底 | 8 套主题

坐席手机号即 PIN，打开 CRM 页面自动检测，无需登录，无需密码。

## 核心功能

1. **自动检测 PIN**：打开 CRM 页面 → CSS 选择器扫号 → 自动存为 PIN
2. **自动检测姓名**：同步抓取业务员姓名，无需手动输入
3. **双模路由**：Ping PC 直连（2s 超时），不可达走云端 `/api/v1/dial`
4. **8 套主题**：右键菜单实时切换，content-script 内联主题
5. **浮动按钮**：CRM 页面注入可拖动的拨号/挂断按钮
6. **右键菜单**：拨号/短信/登记/主题切换/PC状态/PIN显示/同步登记列表
7. **一键登记**：CRM 详情页识别姓名+手机号 → 确认弹窗 → 云中继存储 + CRM 同步
8. **全量同步**：主管从 CRM 来访列表抓取全量记录推入云端

## 文件说明

| 文件 | 角色 |
|------|------|
| `manifest.json` | MV3 清单，匹配 CRM 域名 |
| `background.js` | Service Worker：PIN 管理 + 双模路由 + 登记 API |
| `content-script.js` | 全帧注入：TreeWalker 扫号 + 浮动按钮 + 主题 + 右键菜单 + UI |
| `popup.html` + `popup.js` | 弹窗：服务器配置 + PIN 设置 + 状态查询 |
| `icons/` | 扩展图标 (16/48/128) |
| `AutoDial-API.md` | API 参考文档 |

## 安装

1. Chrome 打开 `chrome://extensions/` → 开启"开发者模式"
2. 点击"加载已解压的扩展程序"
3. 选择 `AutoDial-Extension/` 目录

## 首次配置

1. 点击扩展图标打开弹窗
2. 设置云中继地址（默认 `http://127.0.0.1:35430`，点击"测试"验证连通性）
3. 打开 CRM 页面 → 自动检测坐席手机号为 PIN

## 日常使用

- 打开 CRM → 自动检测坐席号 → 点击拨号按钮 → PC/云端自动路由
- PIN 状态在右键菜单底部可见
- 换号时打开 CRM 重新检测即可

## 拨号反馈

| 提示 | 原因 |
|------|------|
| `✓ 已拨出` | PC 直连成功 |
| `✗ PC已上线，请重试` | 云端发现 PC 在线，应切回本地 |
| `✗ 服务器不可达` | 云中继未启动 |
| `✗ 未检测到坐席手机号` | 请打开 CRM 页面 |

## 右键菜单

- **🎨 切换主题**：8 套主题（默认/暗夜/科技蓝/暖橙/森林绿/优雅紫/商务灰/日落金）
- **📞 拨打 / 💬 发短信**：对当前检测到的客户号码操作
- **📝 一键登记**：CRM 详情页可用，自动识别姓名+手机号
- **🖥 PC 控制**：打开电脑端主界面 / 显示隐藏悬浮窗
- **📋 同步登记列表**：主管抓取 CRM 全量来访记录
- 底部显示 `👤 PIN: 138xxxx` + PC 在线状态

## 一键登记流程

1. 打开 CRM 客户详情页 → 自动识别姓名+手机号
2. 右键悬浮按钮 → 点击「📝 一键登记」
3. 确认弹窗 → 云中继存储 → CRM 同步 → WS 推手机通知
4. 浮动按钮显示「✅ 已登记」

> 需云中继在线（端口 35430）。
