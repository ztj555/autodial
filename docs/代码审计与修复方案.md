## v3 新实现

### 云中继 v3（cloud_relay_v3.py）
- ✅ JWT 认证 WebSocket（35440）
- ✅ REST API（35441）：register/login/refresh/status/dial/result/hangup/sms
- ✅ 双模兼容：同时支持 JWT 和 PIN 握手
- ✅ JWT 路由：user_id 路由（独立于 PIN 路由）
- ✅ 拨号确认机制：dial_ack 更新 dial_results
- ✅ WebSocket 内置 ping/pong（30s/90s）

### 认证模块（auth.py）
- ✅ bcrypt 密码哈希
- ✅ JWT HS256 签名（15min 过期）
- ✅ refresh_token（SHA-256 存储，30天，使用即吊销）
- ✅ 密钥三层管理（env > file > auto）
- ✅ IP + 手机号双维度限流（5次/15min，内存存储）
- ✅ Auto-clean：超过 10000 个 key 时清理 10%

### 数据库（db.py）
- ✅ aiosqlite 异步 SQLite
- ✅ WAL 模式 + PRAGMA user_version 版本管理
- ✅ users / refresh_tokens / devices / audit_log 四表
- ✅ upsert_device（ON CONFLICT 原子操作）
- ✅ verify_and_rotate_refresh_token（验证 + 吊销）
- ✅ cleanup_devices_on_startup（重启清活跃标记）

### Chrome 扩展 v3（background.js）
- ✅ JWT 管理：getToken() → isTokenExpired() → auto-refresh
- ✅ 永不缓存密码（refresh_token 续期）
- ✅ 双模路由：PC 直连优先 → 云端 REST API 兜底
- ✅ PC 重检：15s 间隔，3 次失败切云端
- ✅ 拨号轮询：pollDialResult(req_id, 3000ms)
- ✅ CRM 自检测 → 自动静默续期

## 已知限制

1. 限流器为内存存储，重启后计数清零
2. 邀请码机制已规划但尚未实现
3. 拨号结果暂存仅 100 条（OrderedDict FIFO）
4. 云中继 v3 无 Web 管理面板（只有注册页面）
5. refresh_token 无"全部设备登出"管理界面
