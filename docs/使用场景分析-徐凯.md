# AutoDial 使用指南 v4

> ⚠️ 以下场景分析基于 v3 JWT 时代编写，v4 已简化为 PIN 体系，无登录/密码/SQLite 概念。
> 核心使用流程不变：CRM 检测坐席号 → PC 直连优先 → 云端兜底。端口统一为 35430。

## v3 典型使用场景（历史参考）

### 场景 1：坐席日常拨号（v3 云端模式）

1. 坐席打开 CRM 网页
2. 浏览器扩展自动检测坐席手机号，静默续期 JWT
3. 扩展在 CRM 页面注入浮动拨号按钮
4. 坐席点击号码 → 扩展通过 REST API 发送到云中继 v3
5. 云中继通过 WebSocket 转发到手机
6. 手机自动拨出 → 结果回传 → 扩展轮询确认

### 场景 2：PC 直连模式（v2 保留）

1. PC 端在本机运行，扩展优先检测到本地 PC
2. 所有拨号请求走 PC 端 HTTP API (35432)
3. 延迟更低，不依赖云服务器

### 场景 3：首次注册登录

1. 管理员部署云中继 v3
2. 坐席打开注册页面注册账号
3. Chrome 扩展登录
4. 手机端登录（手机号 + 密码）
5. 后续自动续期，日常无感

## 连接模式选择

| 场景 | 推荐 | 说明 |
|------|------|------|
| PC 开机且本地运行 | PC 直连（自动） | 延迟最低 |
| PC 未开/出门在外 | 云端直连（自动） | 扩展自动切换 |
| 老设备/未升级 | v2 PIN 方式 | 兼容过渡 |

## 扩展自动路由逻辑

```
拨号请求
  ├── isPcAlive() → 127.0.0.1:35432/
  │   ├── YES → PC 直连（v2 API）
  │   └── NO → 检查 JWT
  │        ├── 有 JWT → 云端 REST API (v3)
  │        └── 无 JWT → 提示"请先登录"
  └── PC 掉线后每 15s 重检，恢复后自动切回直连
```

## 故障排查

### 扩展显示"请先登录"
- 点击扩展图标，设置服务器地址并登录
- 确认云中继 v3 正在运行
- 检查服务器地址：应指向云中继 35430 端口（v4 统一端口）

### 拨号后无反应
- v3 模式：检查手机是否通过 JWT 连接了云中继 v3
- v2 模式：检查 PC 端是否运行，防火墙是否放行端口

### 登录失败 / 429 错误
- 15 分钟内超过 5 次失败 → 等待 15 分钟或重启云中继

### 扩展按钮不显示
- 确认当前网页在 manifest.json 的 host_permissions 范围内
- 检查页面是否为 SPA（MutationObserver 会自动检测）
