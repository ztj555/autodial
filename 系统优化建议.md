# 系统架构与配置说明 v4

> 基于实际代码 | v4.0.0 整合版 | 单一 PIN 体系

## 架构演进

```
v2 (旧):  CRM → 浏览器插件 → PC端(35432) → 手机
                                └→ 云中继(35430) → 手机（跨网）

v3 (已废弃):  CRM → 浏览器插件 → 云中继v3(35441 REST) → 云中继v3(35440 WS) → 手机
              ↑ JWT+SQLite+密码栈                 ↑ JWT握手

v4 (当前):  CRM → 浏览器插件 → 优先PC(35432) → 手机
                           └→ 云中继(35430 REST) → 云中继(35430 WS) → 手机
              ↑ X-AutoDial-PIN Header              ↑ 11位PIN强校验
```
```

## 端口体系

| 端口 | 协议 | 认证 | 用途 |
|------|------|------|------|
| **35430** | WS + HTTP | 11 位 PIN | 云中继（WS 中继 + REST API + Web 管理） |
| 35432 | HTTP + WS | 无/PIN | Go PC 端主服务（局域网直连） |
| 35433 | UDP | 无 | LAN 设备发现 |

> v3 端口 35440/35441 已废弃，统一到 35430。

## v4 核心特性

### 安全
- 11 位 PIN 全链路强校验（REST + WS + Go PC 三环节统一）
- PC_CONNECTED 去重 + DUPLICATE_DIAL 5s 并发保护
- IP 频率限制（每分钟 5 次握手尝试）
- 单文件部署，零数据库依赖
- WebSocket 内置 ping/pong（30s/90s 心跳超时）

### 认证流
```
CRM 自动检测坐席手机号 → 存为 PIN → X-AutoDial-PIN Header → 云端 11 位校验
                                                              ├─ 不通过 → INVALID_PIN
                                                              └─ 通过 → 查 PinGroup → 转发
```

### 拨号（火后不管模式）
```
GET /api/v1/dial?number=xxx (Header PIN) → 200 ACCEPTED
手机拨出 → WS dial_result → 记录到日志
```
> 当前阶段采用"火后不管"——拨号指令发出即返回确认，用户从手机屏幕亮起判断拨号状态。

## 部署

单一部署即可：`python cloud_relay_v2.py`（仅依赖 `websockets pystray Pillow`）

## 防火墙要求

- 35430 (TCP)：云中继
- 35432 (TCP)：Go PC 端
- 35433 (UDP)：LAN 设备发现
